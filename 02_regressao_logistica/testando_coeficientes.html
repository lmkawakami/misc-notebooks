Ver o impacto de um fator multiplicativo em uma varivel de entrada e de rudo

Meu pensamento inicial:¶

  • fator multiplicativo afeta coeficientes (ex.: convero de moeda: R$->US$)

*assumindo que a mdia do rudo zero e forando que o rudo no desnormalize os dados

# Funo para aplicar o rudo
def aplly_noise(noise_intensity=1):
  def func(input):
    noise_amplitude = noise_intensity*math.sin(input*math.pi)/4
    noise = np.random.uniform(-noise_amplitude,noise_amplitude)
    return input+noise
  return func

# "intensidade" dos rudos
noise_levels = {
  'noised_a': 0.33333,
  'noised_b': 0.66667,
  'noised_c': 1,
}

# dataframe com entrada pura, com diferentes nveis de rudo, entrada aleatria e sada esperada
df = pd.DataFrame()
df['inputs'] = np.linspace(0,1,10000)
df['noised_a'] = df['inputs'].map(aplly_noise(noise_levels['noised_a']))
df['noised_b'] = df['inputs'].map(aplly_noise(noise_levels['noised_b']))
df['noised_c'] = df['inputs'].map(aplly_noise(noise_levels['noised_c']))
df['random'] = np.random.uniform(0,1, len(df['inputs']))
df['outputs'] = df['inputs'].map(lambda input: 0 if input<0.5 else 1)

# primeiras e ultimas linhas para ilustrar
df.take([0,1,-2,-1])

🤔 Pra que tudo isso?¶

  • inputs: entrada ideal para determinar os outputs
  • noised_X: variaes do inputs com algum rudo. Rudo se intensifica apenas no centro para evitar a "desnormalizao" dos dados (mesmo com o rudo, tudo fica entre 0 e 1)
  • inputs: entrada aleatria que no deve apresentar correlao com outputs
